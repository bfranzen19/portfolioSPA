Why I Want to Work for Automattic

First off, I want to say how much I have enjoyed this application process. I want to work for a company that makes applying a thinking exercise like creating a GET request to get a secret that must be included instead of hitting a button on a website. I love it. The unique application process inspired me to take it a step further and spend an evening getting back to my web development roots creating this page on my single page application portfolio site devoted completely to Automattic.

I first heard about Automattic (beyond WordPress, everyone knows about WordPress) in a LinkedIn video of CEO Matt Mullenweg's TedTalk about The Way We Work. I was instantly intrigued and started reading and researching Automattic. Honestly, the Software Engineer position first caught my eye but once I saw accessibility mentioned in the first bullet point of the JavaScript Engineer posting, I knew I had to apply.

Accessibility is such an easily forgotten standard for everyone who doesn't live with abnormal color vision, people that don't have issues with dyslexia (hence the font used here), or people with other challenges. I want to do better for everyone and make the internet a place that everyone can enjoy.

Automattic is exactly what I'm looking for in a company: a place where ingenuity is embraced, where stellar products are created from treating employees well, and where diversity is not just a lofty goal, but a reality of everyday life (I'm a Honorably Discharge Army Veteran and a woman, by the way). I want to work for a company that's looking for driven, compassionate, independent yet collaborative, supportive, self-motivated, curious unicorns like me and I sincerely hope that I'm exactly who Automattic has been looking for in a JavaScript Engineer.

    • www.triggeraware.com
      This site is a database that allows the user to search by TV or movie title which will return any potential triggers the user should be aware of, categorized by Trigger Type. The database was built using web scraping (Cheerio and Request). The search functionality is made possible by Fuze.js fuzzy search JS library. Each Trigger Type is comprised of several related keyword used in the web scraping search; each Trigger Type is named in a way that should minimize the probability of a user being triggered. Each entry found by scraping is assigned a Trigger Type, depending on the keyword used in the search, when it is entered into the database. Users can also fill out a form to recommend a potentially triggering scene in a TV show or movie. These recommendations go into a separate database until I validate them and move them into the main, searchable database. Validation functionality is behind a user login using bcrypt.
